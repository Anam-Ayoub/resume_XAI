    I. Fuzzy Logic

    Classic Sets vs. Fuzzy Sets

    Classic Sets (Ensembles classiques): In traditional logic, an item completely belongs to a group (a value of 1) or it doesn't belong at all (a value of 0).

    Fuzzy Sets (Ensembles flous): Fuzzy logic allows for partial belonging. An item's membership can be any value in the interval [0, 1].

    Basic Characteristics of Fuzzy Sets

    • Support (Le support): Every item that belongs to the set even a tiny bit ($\mu_A(x) > 0$).
      $$ \text{supp}(A) = \{x \in X, \mu_A(x) > 0\} $$
    • Kernel (Le noyau): Only the items that perfectly and completely belong to the set ($\mu_A(x) = 1$).
      $$ \text{Noyau}(A) = \{x \in X, \mu_A(x) = 1\} $$
    • Height (La hauteur): The highest possible membership score that any item achieves in the whole set.
      $$ H(A) = \sup_{x \in X} \mu_A(x) $$
    • Alpha-cut ($\alpha$-Coupe): A filter that keeps items with a membership score above a specific threshold ($\alpha$).
      $$ A_\alpha = \{x \in X, \mu_A(x) \geq \alpha\} $$
    • Crossover point (Point de croisement): The exact point where an item is right in the middle, with a score of exactly 0.5.
    • Normalized fuzzy sets (Ensembles flous normalisés): A set is considered "normalized" if at least one item in it reaches a perfect score of 1.
      $$ A \text{ est normalisé } \iff \exists x \in X, \mu_A(x) = 1 $$
    • A fuzzy set is convex if and only if its $\alpha$-cuts are convex.

    The Linguistic Variable

    Fuzzy logic computes using everyday words instead of exact numbers to mimic human reasoning.

    • Universe of discourse (L'univers du discours): The complete range of possible values for a given variable.
    • Linguistic values (Valeurs linguistiques): The descriptive words used for different parts of that range (e.g., "cold" or "young").
    • Membership function (Fonction d'appartenance): The mathematical shapes used to map these words to numbers (Triangular, Gaussian, Trapezoidal).

    Triangular Function

    Concept: The triangular membership function is the simplest and most common. It uses three points to form a triangle: a starting point (degree 0), a peak point (degree 1), and an ending point (degree 0).

    Formula:
    $$ \mu(x) = \begin{cases} 0 & x \le a \\ \frac{x - a}{b - a} & a < x \le b \\ \frac{c - x}{c - b} & b < x \le c \\ 0 & x > c \end{cases} $$

    Step-by-step Calculation:

    1. If $x \le a$ or $x \ge c$, the membership degree is 0.
    2. If $x = b$, the degree is 1.
    3. If $a < x < b$, calculate the upward slope: $(x - a) / (b - a)$.
    4. If $b < x < c$, calculate the downward slope: $(c - x) / (c - b)$.

    Trapezoidal Function

    Concept: The trapezoidal function is similar to the triangular one, but instead of a single peak point, it has a flat top where the membership degree remains 1 over an interval.

    Formula:
    $$ \mu(x) = \begin{cases} 0 & x \le a \\ \frac{x - a}{b - a} & a < x \le b \\ 1 & b < x \le c \\ \frac{d - x}{d - c} & c < x \le d \\ 0 & x > d \end{cases} $$

    Step-by-step Calculation:

    1. If $x \le a$ or $x \ge d$, the membership degree is 0.
    2. If $b \le x \le c$, the degree is 1.
    3. If $a < x < b$, calculate the upward slope: $(x - a) / (b - a)$.
    4. If $c < x < d$, calculate the downward slope: $(d - x) / (d - c)$.

    Gaussian Function

    Concept: The Gaussian function creates a smooth, bell-shaped curve. It is infinitely wide but approaches zero as it gets further from the center. It is defined by its center and its width.

    Formula:
    $$ \mu(x) = e^{-\frac{(x - c)^2}{2\sigma^2}} $$

    Step-by-step Calculation:

    1. Calculate the squared distance from center: $(x - c)^2$.
    2. Divide by the spread factor: $(x - c)^2 / (2\sigma^2)$.
    3. Calculate the final degree: $e^{-((x - c)^2 / 2\sigma^2)}$.

    Sigmoid Function

    Concept: The sigmoid function provides a smooth transition from 0 to 1 (or 1 to 0). It is often used for concepts like "very large" or "very small" where the membership degree stays at 1 after a certain point.

    Formula:
    $$ \mu(x) = \frac{1}{1 + e^{-a(x - c)}} $$

    Step-by-step Calculation:

    1. Calculate the distance multiplied by slope: $-a(x - c)$.
    2. Apply the exponential function: $e^{-a(x - c)}$.
    3. Calculate the final degree: $1 / (1 + e^{-a(x - c)})$.

    Fuzzy Logic Operators (Zadeh's Formalism)

    When combining fuzzy sets, we use specific mathematical rules:

    • Logical AND (ET logique): Represents the intersection by taking the smallest value: $\min(a, b)$.
    • Logical OR (OU logique): Represents the union by taking the largest value: $\max(a, b)$.
    • Logical NOT (NON logique): Represents the opposite (le complément), calculated as $1 - a$.

    II. Inference Systems

    1. Classic Inference Systems

    These systems use strict Boolean logic (0=False, 1=True) to deduce conclusions.

    • Knowledge Base (Base de connaissances): Comprises the Database (known facts) and the Rule Base (IF-THEN rules).
    • Inference Engine (Moteur d'inférence): Searches for and applies rules to data.
    • Methods: Forward Chaining (Facts → Conclusion) and Backward Chaining (Goal → Facts).
    • Pros & Cons: Simple, fast, but cannot handle uncertainty and has abrupt boundaries.

    2. Fuzzy Inference Systems (FIS)

    Handle uncertainty by using linguistic variables instead of strict 0s and 1s. Formally defined by (X, Y, F, R, M).

    1. Fuzzification: Converting a precise number into a membership degree.
    2. Inference: Applying linguistic rules (e.g., "IF Temp is Cold THEN Fan is Slow").
    3. Defuzzification: Converting the final fuzzy result back into a precise number.

    3. Fuzzy Inference Models

    • Mamdani Model (Le modèle Mamdani): The conclusion of each rule is a fuzzy set. Defuzzification uses the weighted average of the centers:
      $$ z = \frac{\sum (w_i \cdot C_i)}{\sum w_i} $$
      ($w_i$ = rule activation degree, $C_i$ = center of output triangle).
    • Sugeno Model (Le modèle Sugeno): The conclusion is a strict mathematical function, great for optimization.
      $$ z = \frac{\sum (w_i \cdot z_i)}{\sum w_i} $$
    • Tsukamoto Model (Modèle Tsukamoto): Output sets must be strictly monotonic. Output is calculated using specific values on the curve.

    III. Fuzzy Clustering

    Review: Classic / Hard Clustering

    Data points belong to exactly one cluster. $u_{ij} \in \{0, 1\}$.

    Limitations: Forces abrupt assignments, causes information loss, and is sensitive to noise.

    Mathematical Formalism

    A single data point can belong to multiple clusters at the same time based on a membership degree.

    • Constraints:
      $$ 0 \leq u_{ij} \leq 1 \quad \text{and} \quad \sum_{j=1}^{c} u_{ij} = 1 $$
    • FCM Objective Function:
      $$ J_m = \sum_{i=1}^{n}\sum_{j=1}^{c} u_{ij}^m ||x_i - c_j||^2 $$
      Where $m$ is the fuzziness coefficient.

    FCM Algorithm Steps

    1. Initialize membership matrix $U$.
    2. Calculate Centers:
      $$ c_j = \frac{\sum_{i=1}^{n} u_{ij}^m x_i}{\sum_{i=1}^{n} u_{ij}^m} $$
    3. Update Memberships:
      $$ u_{ij} = \frac{1}{\sum_{k=1}^{c} \left( \frac{||x_i - c_j||}{||x_i - c_k||} \right)^{\frac{2}{m-1}}} $$
    4. Repeat until convergence (difference $< \epsilon$).

    FCM Python Implementation

    import numpy as np

# --- 1. DATA INITIALIZATION ---
X = np.array([1, 2, 5, 8], dtype=float)
n = len(X)
c = 2 # Number of clusters
m = 2 # Fuzziness parameter
epsilon = 0.1 # Stopping criteria
max_iter = 100

# Initial membership matrix
U = np.array([
    [0.8, 0.2],
    [0.7, 0.3],
    [0.3, 0.7],
    [0.2, 0.8]
], dtype=float)

# --- 2. MATH FUNCTIONS ---
def calcul_centres(X, U, m):
    um = U ** m
    # Weighted average
    centres = (um.T @ X) / np.sum(um.T, axis=1) 
    return centres

def update_U(X, centres, m):
    n, c = len(X), len(centres)
    U_new = np.zeros((n, c))
    for i in range(n):
        for j in range(c):
            somme = 0
            for k in range(c):
                d1 = abs(X[i] - centres[j])
                d2 = abs(X[i] - centres[k])
                if d2 == 0:
                    d2 = 1e-6
                somme += (d1 / d2) ** (2 / (m - 1))
            U_new[i, j] = 1 / somme
    return U_new

# --- 3. ALGORITHM LOOP ---
iteration = 0
while iteration < max_iter:
    iteration += 1
    centres = calcul_centres(X, U, m)
    U_new = update_U(X, centres, m)
    
    if np.linalg.norm(U_new - U) < epsilon:
        break
        
    U = U_new

    IV. Adaptive Neuro-Fuzzy Inference System (ANFIS)

    Why ANFIS?

    Standard Neural Networks (ANN) are powerful but act as a "black box" that cannot explain decisions. Standard Fuzzy Logic is interpretable but requires manual rule creation. ANFIS combines both: it learns parameters automatically from data while generating clear, interpretable IF-THEN rules.

    Architecture (The 5 Layers)

    1. Layer 1 (Fuzzification): Calculates membership degree.
    2. Layer 2 (Rule Activation): Multiplies inputs to find the activation strength of the rule.
    3. Layer 3 (Normalization): Calculates the relative weight of the rule.
    4. Layer 4 (Rule Output): Applies the mathematical Sugeno function.
    5. Layer 5 (Final Output): Calculates final global output (Defuzzification).

    Comparison & Applications

    ANN vs ANFIS: ANN says "I predict, but I cannot explain", whereas ANFIS says "I predict AND I explain using rules".

    Pros & Cons: Highly precise and explainable, but computationally expensive and sensitive to data.

    Example (Credit Risk): "IF income is low AND debt is high THEN credit = refused". Used in finance, medical diagnosis, smart grids, robotics, and XAI.

    V. Fuzzy Quantifiers

    Why use them?

    Why use them? (Pourquoi les utiliser ?) Classic rules (like "IF CPU is high") are too rigid and lack explanation. Quantifiers (like "IF most indicators are high") make the rules much more natural, flexible, and interpretable.

    Formal Definition (Définition formelle): A fuzzy quantifier is a mathematical function that takes a satisfied proportion ($p$) and converts it into a degree of truth ($Q(p)$).

    $$ Q: [0, 1] \rightarrow [0, 1] $$

    Mathematical Example

    To calculate the exact degree of truth for the word "most" (la plupart), we use this function:

    $$ Q_{\text{most}}(p) = \begin{cases} 0 & \text{for } p < 0.5 \\ 2p - 1 & \text{for } 0.5 \leq p \leq 1 \end{cases} $$

    Interpretation: If less than 50% of conditions are met, "most" is 0% true. Once you pass 50%, the truth degree scales up progressively until it hits 100% truth at $p=1$.

    Types of Fuzzy Quantifiers

    • Monotonic (Quantificateurs monotones): Functions that only go in one direction (strictly increasing like "a lot" or strictly decreasing like "few"). Logically coherent, but cannot cover specific target concepts.
    • Non-monotonic (Quantificateurs non monotones): Functions centered on a specific target value, using triangular or Gaussian shapes (e.g., "about 5"). Great for precise concepts.
    • Regular (Quantificateurs réguliers de Zadeh): These must strictly start at zero and end at exactly one ($Q(0)=0$ and $Q(1)=1$). Examples: "most", "a lot".
    • Non-regular (Quantificateurs non réguliers): Do not respect Zadeh's strict boundaries. Flexible for real-world situations.
    • Linguistic (Quantificateurs linguistiques flous): Natural language expressions modeled directly by fuzzy functions. Extremely natural but highly subjective.

    VI. Fuzzy Decision Trees

    1. Introduction & Review (Classic Trees)

    • Classic Trees: Supervised learning models making hierarchical decisions via rules. They consist of a root node, branches, and leaves, where each node applies a strict, precise binary test (e.g., Age > 30). 
              Age > 30 ?
        /        \
      Yes         No
      /            \
 High Income?     Refuse
    /      \
  Yes      No
Accept    Refuse
    • Fuzzy Decision Trees: Extend classic trees using fuzzy logic to handle uncertainty and imprecision. A single data point can belong partially to multiple classes.
    • Construction of a Classic Tree: Uses algorithms like ID3, C4.5, and CART to find the best attributes to separate the data based on calculation criteria:
      • Entropy: Calculates uncertainty: $H(S) = - \sum_{i=1}^{n} p_i \log_2(p_i)$
      • Information Gain: Calculates the reduction in uncertainty to pick the best attribute: $Gain(S,A) = H(S) - \sum_{v \in Values(A)} ( \frac{|S_v|}{|S|} ) H(S_v)$

    2. Why Use a Fuzzy Decision Tree?

    Classic trees have several limitations:

    • Strict Boundaries: Rigid cutoffs are not realistic (e.g., a tiny change leads to opposite decisions).
    • No Linguistic Terms: They manipulate exact numbers instead of human concepts like "low salary".
    • Poor Uncertainty Handling: Highly unstable to small data variations.
    • Rule Explosion & Overfitting: Deep trees memorize training data but fail on new clients.

    The Fuzzy Solution: Fuzzy trees replace strict bounds with membership degrees, making decisions progressive instead of binary.

    3. Construction of a Fuzzy Decision Tree

    The goal is to build the tree using fuzzy membership degrees, calculate fuzzy entropies, and generate fuzzy rules.

    • Fuzzy Sets: Instead of a strict boundary, variables are divided into fuzzy sets like "Low", "Medium", and "High".
    • Fuzzy Entropy: Similar to classic entropy, but uses fuzzy probabilities. To find the probability, you sum the membership degrees for each class instead of counting whole units.

    4. Rule Generation

    • Fuzzy Weights: The weight of each fuzzy branch is calculated by summing the membership degrees.
    • Fuzzy Information Gain: Determines which attribute to place at the top of the tree. 
                Salary
        /   |   \
     Low Medium High
      |     |     |
   Refuse Mixed Accept
    • Rule Generation Principle: A rule is created by taking the fuzzified values, selecting the dominant class, and translating the path into natural linguistic terms.
    Examples of generated rules:

    Rule 1: IF Salary is Low AND Debt is High THEN Decision = Refuse
    Rule 2: IF Salary is Medium AND Age is Young AND Debt is Medium THEN Decision = Accept
    Rule 3: IF Salary is High AND Age is Medium AND Debt is Low THEN Decision = Accept

    5. Conclusion

    • Main Advantages: Better uncertainty management, progressive decisions, greater robustness, and rules that map closely to natural human reasoning.
    • Applications: Expert systems, finance, medicine, and general decision support.

    I. Logique Floue

    Ensembles Classiques vs. Ensembles Flous

    Ensembles classiques : Dans la logique traditionnelle, un élément appartient complètement à un groupe (une valeur de 1) ou n'y appartient pas du tout (une valeur de 0).

    Ensembles flous : La logique floue permet une appartenance partielle. L'appartenance d'un élément peut être n'importe quelle valeur dans l'intervalle [0, 1].

    Caractéristiques de base des ensembles flous

    • Le support : Tout élément qui appartient à l'ensemble même un tout petit peu ($\mu_A(x) > 0$).
      $$ \text{supp}(A) = \{x \in X, \mu_A(x) > 0\} $$
    • Le noyau : Uniquement les éléments qui appartiennent parfaitement et complètement à l'ensemble ($\mu_A(x) = 1$).
      $$ \text{Noyau}(A) = \{x \in X, \mu_A(x) = 1\} $$
    • La hauteur : Le plus haut score d'appartenance possible atteint dans tout l'ensemble.
      $$ H(A) = \sup_{x \in X} \mu_A(x) $$
    • $\alpha$-Coupe : Un filtre qui conserve uniquement les éléments ayant un score d'appartenance supérieur à un seuil spécifique ($\alpha$).
      $$ A_\alpha = \{x \in X, \mu_A(x) \geq \alpha\} $$
    • Point de croisement : Le point exact où un élément se trouve juste au milieu, avec un score de exactement 0.5.
    • Ensembles flous normalisés : Un ensemble est considéré comme "normalisé" si au moins l'un de ses éléments atteint un score parfait de 1.
      $$ A \text{ est normalisé } \iff \exists x \in X, \mu_A(x) = 1 $$
    • Un ensemble flou est convexe si et seulement si ses $\alpha$-coupes sont convexes.

    La Variable Linguistique

    La logique floue calcule en utilisant des mots de la vie courante au lieu de nombres exacts pour imiter le raisonnement humain.

    • L'univers du discours : La plage complète des valeurs possibles pour une variable donnée.
    • Valeurs linguistiques : Les mots descriptifs utilisés pour différentes parties de cette plage (ex: "froid" ou "jeune").
    • Fonction d'appartenance : Les formes mathématiques utilisées pour mapper ces mots à des nombres (Triangulaire, Gaussienne, Trapézoïdale).

    Fonction Triangulaire

    Concept : La fonction d'appartenance triangulaire est la plus simple et la plus courante. Elle utilise trois points pour former un triangle : un point de départ (degré 0), un sommet (degré 1) et un point de fin (degré 0).

    Formule :
    $$ \mu(x) = \begin{cases} 0 & x \le a \\ \frac{x - a}{b - a} & a < x \le b \\ \frac{c - x}{c - b} & b < x \le c \\ 0 & x > c \end{cases} $$

    Calcul étape par étape :

    1. Si $x \le a$ ou $x \ge c$, le degré d'appartenance est 0.
    2. Si $x = b$, le degré est 1.
    3. Si $a < x < b$, calculez la pente ascendante : $(x - a) / (b - a)$.
    4. Si $b < x < c$, calculez la pente descendante : $(c - x) / (c - b)$.

    Fonction Trapézoïdale

    Concept : La fonction trapézoïdale est similaire à la fonction triangulaire, mais au lieu d'un seul point au sommet, elle possède un plateau où le degré d'appartenance reste à 1 sur un intervalle donné.

    Formule :
    $$ \mu(x) = \begin{cases} 0 & x \le a \\ \frac{x - a}{b - a} & a < x \le b \\ 1 & b < x \le c \\ \frac{d - x}{d - c} & c < x \le d \\ 0 & x > d \end{cases} $$

    Calcul étape par étape :

    1. Si $x \le a$ ou $x \ge d$, l'appartenance est 0.
    2. Si $b \le x \le c$, l'appartenance est 1.
    3. Si $a < x < b$, calculez la pente ascendante : $(x - a) / (b - a)$.
    4. Si $c < x < d$, calculez la pente descendante : $(d - x) / (d - c)$.

    Fonction Gaussienne

    Concept : La fonction gaussienne crée une courbe douce en forme de cloche. Elle est infiniment large mais s'approche de zéro en s'éloignant du centre. Elle est définie par son centre et sa largeur.

    Formule :
    $$ \mu(x) = e^{-\frac{(x - c)^2}{2\sigma^2}} $$

    Calcul étape par étape :

    1. Calculez la distance au centre au carré : $(x - c)^2$.
    2. Divisez par le facteur d'écart : $(x - c)^2 / (2\sigma^2)$.
    3. Calculez le degré final : $e^{-((x - c)^2 / 2\sigma^2)}$.

    Fonction Sigmoïde

    Concept : La fonction sigmoïde offre une transition douce de 0 à 1 (ou de 1 à 0). Elle est souvent utilisée pour des concepts tels que "très grand" ou "très petit", où le degré d'appartenance reste à 1 après un certain point.

    Formule :
    $$ \mu(x) = \frac{1}{1 + e^{-a(x - c)}} $$

    Calcul étape par étape :

    1. Calculez la distance multipliée par la pente : $-a(x - c)$.
    2. Appliquez la fonction exponentielle : $e^{-a(x - c)}$.
    3. Calculez le degré final : $1 / (1 + e^{-a(x - c)})$.

    Opérateurs à Logique Floue (Formalisme de Zadeh)

    Lors de la combinaison d'ensembles flous, nous utilisons des règles mathématiques spécifiques :

    • ET logique : Représente l'intersection en prenant la plus petite valeur : $\min(a, b)$.
    • OU logique : Représente l'union en prenant la plus grande valeur : $\max(a, b)$.
    • NON logique : Représente le complémentaire, calculé comme $1 - a$.

    II. Systèmes d'Inférence

    1. Systèmes d'Inférence Classiques

    Ces systèmes utilisent la logique booléenne stricte (0=Faux, 1=Vrai) pour déduire des conclusions.

    • Base de connaissances : Comprend la Base de données (faits connus) et la Base de règles (règles SI-ALORS).
    • Moteur d'inférence : Cherche et applique les règles aux données.
    • Méthodes : Chaînage avant (Faits → Conclusion) et Chaînage arrière (But → Faits).
    • Avantages & Inconvénients : Simple et rapide, mais ne peut pas gérer l'incertitude et possède des frontières brutales.

    2. Systèmes d'Inférence Flous (FIS)

    Gèrent l'incertitude en utilisant des variables linguistiques au lieu de 0 et 1 stricts. Définis formellement par (X, Y, F, R, M).

    1. Fuzzification : Conversion d'un nombre précis en un degré d'appartenance.
    2. Inférence : Application des règles linguistiques (ex: "SI Température est Froide ALORS Ventilateur est Lent").
    3. Défuzzification : Conversion du résultat flou final en un nombre précis.

    3. Modèles d'Inférence Floue

    • Le modèle Mamdani : La conclusion de chaque règle est un ensemble flou. La défuzzification utilise la moyenne pondérée des centres :
      $$ z = \frac{\sum (w_i \cdot C_i)}{\sum w_i} $$
      ($w_i$ = degré d'activation de la règle, $C_i$ = centre du triangle de sortie).
    • Le modèle Sugeno : La conclusion est une fonction mathématique stricte, idéale pour l'optimisation.
      $$ z = \frac{\sum (w_i \cdot z_i)}{\sum w_i} $$
    • Modèle Tsukamoto : Les ensembles de sortie doivent être strictement monotones. La sortie est calculée en utilisant des valeurs spécifiques sur la courbe.

    III. Clustering Flou

    Rappel : Clustering Classique / Dur

    Les points de données appartiennent à exactement un seul cluster. $u_{ij} \in \{0, 1\}$.

    Limites : Force des affectations brutales, entraîne une perte d'information et est sensible au bruit.

    Formalisme Mathématique

    Un seul point de données peut appartenir à plusieurs clusters en même temps selon un degré d'appartenance.

    • Contraintes :
      $$ 0 \leq u_{ij} \leq 1 \quad \text{et} \quad \sum_{j=1}^{c} u_{ij} = 1 $$
    • Fonction Objectif FCM :
      $$ J_m = \sum_{i=1}^{n}\sum_{j=1}^{c} u_{ij}^m ||x_i - c_j||^2 $$
      Où $m$ est le coefficient de flou.

    Étapes de l'Algorithme FCM

    1. Initialiser la matrice d'appartenance $U$.
    2. Calculer les centres :
      $$ c_j = \frac{\sum_{i=1}^{n} u_{ij}^m x_i}{\sum_{i=1}^{n} u_{ij}^m} $$
    3. Mettre à jour les appartenances :
      $$ u_{ij} = \frac{1}{\sum_{k=1}^{c} \left( \frac{||x_i - c_j||}{||x_i - c_k||} \right)^{\frac{2}{m-1}}} $$
    4. Répéter jusqu'à convergence (différence $< \epsilon$).

    Implémentation Python FCM

    import numpy as np

# --- 1. INITIALISATION DES DONNÉES ---
X = np.array([1, 2, 5, 8], dtype=float)
n = len(X)
c = 2 # Nombre de clusters
m = 2 # Paramètre de flou
epsilon = 0.1 # Critère d'arrêt
max_iter = 100

# Matrice d'appartenance initiale
U = np.array([
    [0.8, 0.2],
    [0.7, 0.3],
    [0.3, 0.7],
    [0.2, 0.8]
], dtype=float)

# --- 2. FONCTIONS DE CALCUL MATHÉMATIQUE ---
def calcul_centres(X, U, m):
    um = U ** m
    # Moyenne pondérée
    centres = (um.T @ X) / np.sum(um.T, axis=1) 
    return centres

def update_U(X, centres, m):
    n, c = len(X), len(centres)
    U_new = np.zeros((n, c))
    for i in range(n):
        for j in range(c):
            somme = 0
            for k in range(c):
                d1 = abs(X[i] - centres[j])
                d2 = abs(X[i] - centres[k])
                if d2 == 0:
                    d2 = 1e-6
                somme += (d1 / d2) ** (2 / (m - 1))
            U_new[i, j] = 1 / somme
    return U_new

# --- 3. BOUCLE PRINCIPALE ---
iteration = 0
while iteration < max_iter:
    iteration += 1
    centres = calcul_centres(X, U, m)
    U_new = update_U(X, centres, m)
    
    if np.linalg.norm(U_new - U) < epsilon:
        break
        
    U = U_new

    IV. Système d'Inférence Neuro-Flou Adaptatif (ANFIS)

    Pourquoi ANFIS ?

    Les réseaux de neurones (ANN) sont puissants mais agissent comme une "boîte noire" qui ne peut pas expliquer ses décisions. La logique floue est interprétable mais nécessite la création manuelle de règles. ANFIS combine les deux : il apprend les paramètres automatiquement à partir des données tout en générant des règles SI-ALORS claires et interprétables.

    Architecture (Les 5 Couches)

    1. Couche 1 (Fuzzification) : Calcule le degré d'appartenance.
    2. Couche 2 (Activation des règles) : Multiplie les entrées pour trouver la force d'activation de la règle.
    3. Couche 3 (Normalisation) : Calcule le poids relatif de la règle.
    4. Couche 4 (Sortie des règles) : Applique la fonction mathématique de Sugeno.
    5. Couche 5 (Sortie finale) : Calcule la sortie globale finale (Défuzzification).

    Comparaison & Applications

    ANN vs ANFIS : ANN dit "Je prédis, mais je ne peux pas expliquer", alors qu'ANFIS dit "Je prédis ET j'explique en utilisant des règles".

    Avantages & Inconvénients : Très précis et explicable, mais coûteux en calcul et sensible aux données.

    Exemple (Risque de crédit) : "SI revenu est faible ET dette est élevée ALORS crédit = refusé". Utilisé en finance, diagnostic médical, smart grids, robotique et XAI.

    V. Quantificateurs Flous

    Pourquoi les utiliser ?

    Les règles classiques (comme "SI CPU est élevé") sont trop rigides et manquent d'explication. Les quantificateurs (comme "SI la plupart des indicateurs sont élevés") rendent les règles beaucoup plus naturelles, flexibles et interprétables.

    Définition formelle : Un quantificateur flou est une fonction mathématique qui prend une proportion satisfaite ($p$) et la convertit en un degré de vérité ($Q(p)$).

    $$ Q: [0, 1] \rightarrow [0, 1] $$

    Exemple mathématique

    Pour calculer le degré de vérité exact du mot "la plupart", nous utilisons cette fonction :

    $$ Q_{\text{most}}(p) = \begin{cases} 0 & \text{pour } p < 0.5 \\ 2p - 1 & \text{pour } 0.5 \leq p \leq 1 \end{cases} $$

    Interprétation : Si moins de 50% des conditions sont remplies, "la plupart" est vrai à 0%. Une fois les 50% dépassés, le degré de vérité augmente progressivement jusqu'à atteindre 100% à $p=1$.

    Types de quantificateurs flous

    • Quantificateurs monotones : Fonctions qui ne vont que dans une seule direction (strictement croissantes comme "beaucoup" ou strictement décroissantes comme "peu"). Logiquement cohérents, mais ne peuvent pas couvrir des concepts cibles spécifiques.
    • Quantificateurs non monotones : Fonctions centrées sur une valeur cible spécifique, utilisant des formes triangulaires ou gaussiennes (ex: "environ 5"). Idéal pour des concepts précis.
    • Quantificateurs réguliers de Zadeh : Doivent strictement commencer à zéro et se terminer à exactement un ($Q(0)=0$ et $Q(1)=1$). Exemples : "la plupart", "beaucoup".
    • Quantificateurs non réguliers : Ne respectent pas les frontières strictes de Zadeh. Très flexibles pour des situations du monde réel.
    • Quantificateurs linguistiques flous : Expressions du langage naturel modélisées directement par des fonctions floues. Extrêmement naturels mais très subjectifs.

    VI. Arbres de Décision Flous

    1. Introduction & Rappel (Arbres Classiques)

    • Arbres classiques : Modèles d'apprentissage supervisé prenant des décisions hiérarchiques via des règles. Ils appliquent un test binaire strict et précis (ex : Age > 30). 
              Age > 30 ?
        /        \
      Oui        Non
      /            \
Revenu élevé ?    Refus
    /      \
  Oui      Non
Accepte   Refus
    • Arbres de Décision Flous : Étendent les arbres classiques en utilisant la logique floue pour gérer l'incertitude. Un point de données peut appartenir partiellement à plusieurs classes.
    • Construction d'un Arbre Classique : Utilise des algorithmes comme ID3, C4.5 et CART pour séparer les données selon des critères mathématiques :
      • Entropie : Calcule l'incertitude : $H(S) = - \sum_{i=1}^{n} p_i \log_2(p_i)$
      • Gain d'information : Réduction de l'incertitude pour choisir le meilleur attribut : $Gain(S,A) = H(S) - \sum_{v \in Valeurs(A)} ( \frac{|S_v|}{|S|} ) H(S_v)$

    2. Pourquoi utiliser un arbre de décision flou ?

    Les arbres classiques ont plusieurs limites :

    • Frontières de décision brutales : Les limites strictes ne sont pas réalistes.
    • Absence de notion linguistique : Manipulation de nombres exacts au lieu de concepts humains (ex : "salaire faible").
    • Mauvaise gestion de l'incertitude : Très instable face aux petites variations.
    • Surapprentissage : Arbres profonds qui mémorisent les données d'entraînement mais échouent sur de nouveaux clients.

    La Solution Floue : Remplacer les limites strictes par des degrés d'appartenance, rendant les décisions progressives.

    3. Construction d'un Arbre de Décision Flou

    L'objectif est de construire l'arbre en utilisant des degrés d'appartenance flous, calculer les entropies floues et générer des règles floues.

    • Ensembles flous : Les variables sont divisées en ensembles flous (ex: "Faible", "Moyen", "Élevé").
    • Entropie Floue : Utilise des probabilités floues (en sommant les degrés d'appartenance pour chaque classe).

    4. Génération des règles

    • Poids flous des ensembles : Le poids de chaque branche est calculé en sommant les degrés d'appartenance.
    • Gain d'Information Flou : Détermine quel attribut placer au sommet de l'arbre. 
               Salaire
       /    |    \
  Faible  Moyen  Élevé
    |       |      |
 Refuser  Mixte  Accorder
    • Principe de génération : Une règle est créée en prenant les valeurs fuzzifiées, sélectionnant la classe dominante et en traduisant le chemin en termes linguistiques naturels.
    Exemples de règles générées :

    Règle 1 : SI Salaire est Faible ET Dette est Élevée ALORS Décision = Refuser
    Règle 2 : SI Salaire est Moyen ET Age est Jeune ET Dette est Moyenne ALORS Décision = Accorder
    Règle 3 : SI Salaire est Élevé ET Age est Moyen ET Dette est Faible ALORS Décision = Accorder

    5. Conclusion (Arbres flous)

    • Avantages principaux : Meilleure gestion de l'incertitude, décisions progressives, robustesse améliorée et règles proches du raisonnement humain.
    • Domaines d'application : Systèmes experts, finance, médecine et aide à la décision.
